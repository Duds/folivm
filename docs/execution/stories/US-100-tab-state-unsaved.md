# US-100 — Tab state and unsaved-change handling

**As an** author,  
**I want** each tab to track its content and unsaved state  
**so that** closing a tab with unsaved changes prompts me to save.

## Acceptance Criteria

- [ ] Document content is cached per tab; switching tabs swaps content without reload
- [ ] Tab shows unsaved indicator (e.g. dot) when content has changed
- [ ] Closing tab with unsaved changes: prompt to Save / Don't save / Cancel
