# US-070 — Documented frontmatter schema

**As an** author,  
**I want** documented frontmatter schema  
**so that** I know what metadata is valid and how to use it.

## Acceptance Criteria

- [ ] Folivm YAML frontmatter schema is documented (required/optional fields)
- [ ] Field semantics are explained (title, created, updated, etc.)
- [ ] Format documentation is versioned and referenced from README
- [ ] A third party can create valid frontmatter using only the documentation
