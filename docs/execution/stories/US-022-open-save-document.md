# US-022 — Open and save document

**As an** author,  
**I want to** open a document from a project and save changes  
**so that** my edits persist.

## Acceptance Criteria

- [ ] I can open a document; content loads into the editor
- [ ] I can save (shortcut or menu); changes are written to the file
- [ ] Saved file remains valid Pandoc Markdown with YAML frontmatter
- [ ] Reopening the document shows my saved content
- [ ] Save path is within the project (working/ or deliverables/)
